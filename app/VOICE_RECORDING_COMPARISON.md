# 语音录制方案对比

## 方案一：SpeechRecognizer方案（VoiceRecordingService）

### 工作原理
使用Android系统的SpeechRecognizer API进行实时语音识别，识别完成后快速重启继续监听。

### 优点
- ✅ 系统级API，稳定可靠
- ✅ 直接返回识别文本，无需额外处理
- ✅ 支持多语言（10种语言）
- ✅ 资源占用相对较低
- ✅ 无需额外权限

### 缺点
- ❌ 存在重启间隙（50ms），可能漏记录
- ❌ 依赖系统识别服务
- ❌ 可能出现"假就绪"状态
- ❌ 需要频繁重新初始化（每10次）
- ❌ Xiaomi等设备兼容性问题

### 技术参数
- **重启延迟**: 50ms
- **重新初始化间隔**: 每10次识别
- **假就绪检测**: 5秒超时
- **连续错误阈值**: 5次

### 适用场景
- 对实时性要求不高
- 可以接受少量漏记录
- 希望降低资源占用
- 需要快速部署

## 方案二：AudioRecord连续录制方案（ContinuousVoiceRecordingService）

### 工作原理
使用AudioRecord持续录制原始音频流，按时间段分割（30秒/段），段与段之间保持5秒重叠，只保存和识别包含有效语音的片段。

### 优点
- ✅ 零漏记录（连续录制）
- ✅ 5秒重叠确保不丢失边界语音
- ✅ 智能过滤静音片段
- ✅ 完全控制录制流程
- ✅ 可自由选择识别引擎

### 缺点
- ❌ 资源占用较高（持续录制）
- ❌ 需要额外的音频处理
- ❌ 需要实现语音识别接口
- ❌ 存储空间占用较大
- ❌ 电池消耗较高

### 技术参数
- **采样率**: 16000 Hz
- **音频格式**: PCM 16-bit
- **段长度**: 30秒
- **重叠时长**: 5秒
- **最小音频电平**: 500
- **静音阈值**: 2秒
- **有效语音判定**: >10%样本有声音

### 工作流程
```
1. 启动AudioRecord持续录制
2. 实时检测音频电平
3. 累积音频数据到段缓冲区
4. 达到30秒或检测到2秒静音后：
   - 检查是否包含有效语音
   - 如果有效，保存为PCM文件
   - 调用识别引擎处理
5. 重置缓冲区，添加5秒重叠数据
6. 继续录制下一段
```

### 音频段处理
```kotlin
// 段结构

// 示例时间线
```

## 方案对比表

| 特性 | 方案一（SpeechRecognizer） | 方案二（AudioRecord） |
|------|---------------------------|---------------------|
| **漏记录风险** | 低（50ms间隙） | 无（连续录制） |
| **资源占用** | 低 | 高 |
| **电池消耗** | 低 | 高 |
| **存储占用** | 极低（仅文本） | 高（音频文件） |
| **实时性** | 高（即时识别） | 中（段处理） |
| **准确性** | 高（系统API） | 取决于识别引擎 |
| **稳定性** | 中（需优化） | 高（完全控制） |
| **实现复杂度** | 低 | 高 |
| **网络依赖** | 可能需要 | 取决于识别方案 |
| **离线支持** | 部分支持 | 完全支持 |

## 推荐使用场景

### 使用方案一（SpeechRecognizer）如果：
- ✅ 日常会议记录
- ✅ 课堂笔记
- ✅ 个人日记
- ✅ 一般对话记录
- ✅ 移动设备使用

### 使用方案二（AudioRecord）如果：
- ✅ 法律取证
- ✅ 医疗记录
- ✅ 重要会议
- ✅ 专业采访
- ✅ 需要后期审核

---

## 混合方案建议

可以考虑实现一个混合方案：
1. 默认使用方案一（低功耗）
2. 重要场景切换到方案二（高准确）
3. 在UI中提供切换选项
4. 根据电量自动调整

```kotlin
// 伪代码
when {
    isImportantMode -> useContinuousRecording()
    batteryLevel < 20 -> useSpeechRecognizer()
    else -> userPreference()
}
```

## 实现建议

### 方案一优化
1. 进一步减少重启延迟（尝试20ms）
2. 改进假就绪检测算法
3. 添加音频缓冲区保存最后1秒
4. 实现智能重启策略

### 方案二优化
1. 实现自适应段长度
2. 改进语音活动检测（VAD）
3. 添加降噪预处理
4. 实现增量识别
5. 优化存储管理

---

## 结论

- **方案一**适合大多数日常使用场景，平衡了功能和性能
- **方案二**适合对准确性要求极高的专业场景
- 建议在应用中同时实现两种方案，让用户根据需求选择
- 可以考虑实现混合方案，在不同场景自动切换
